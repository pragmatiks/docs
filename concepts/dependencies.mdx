---
title: "Dependencies"
description: "Resource dependency management"
---

Resources can depend on other resources. Pragmatiks ensures dependencies are satisfied before processing dependent resources.

## Declaring Dependencies

Use `depends_on` to declare dependencies:

```yaml
provider: gcp
resource: bigquery-dataset
name: analytics
depends_on:
  - gcp/storage/data-lake
config:
  location: EU
```

## Dependency Resolution

When you apply a resource with dependencies:

1. Pragmatiks checks if all dependencies exist
2. Dependencies must be in READY state
3. If dependencies aren't ready, the resource waits

## Field References

Reference outputs from dependencies in your configuration:

```yaml
provider: gcp
resource: bigquery-dataset
name: analytics
depends_on:
  - gcp/storage/data-lake
config:
  source_bucket: "${gcp/storage/data-lake.outputs.name}"
```

## Dependency Cycles

Circular dependencies are not allowed. If A depends on B, B cannot depend on A.
